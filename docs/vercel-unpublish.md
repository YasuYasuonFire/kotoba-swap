## 目的

このリポジトリの Vercel デプロイを「停止（連携解除）」し、サイトを「非公開（アクセス不可/認証必須）」にします。

この作業は **Vercel ダッシュボード側の操作が必須**です（リポジトリの変更だけでは Git 連携を解除できません）。

---

## 1) いますぐサイトを非公開にする（推奨：Vercel側で停止）

最短で完全に非公開にしたい場合は、以下のどちらかを実施してください。

### A. プロジェクトを削除（完全に公開停止）

1. Vercel ダッシュボード → 対象プロジェクト
2. `Settings` → `General`
3. `Delete Project`（削除）

これで **本番URLもプレビューURLも**基本的にアクセスできなくなります。

### B. Git 連携を解除（自動デプロイ停止）

1. Vercel ダッシュボード → 対象プロジェクト
2. `Settings` → `Git`
3. `Connected Git Repository` の `Disconnect`（リポジトリ切断）

これで Git push などによる **自動デプロイが止まります**。

---

## 2) 「もし再デプロイされても」本番を非公開にする（リポジトリ側の対策）

このリポジトリには、本番（Vercel Production）で Basic 認証を強制する `web/middleware.ts` を追加しています。

### 設定する環境変数（Vercel Production）

- `SITE_BASIC_AUTH_USER`
- `SITE_BASIC_AUTH_PASS`

未設定のまま本番に上がった場合は、意図せず公開しないように **503 でブロック**します。

---

## 3) 追加の確認（おすすめ）

- Vercel の `Deployments` に新しいデプロイが作成されないこと
- GitHub 側の Vercel アプリ連携が不要なら解除（組織ポリシーに従ってください）
